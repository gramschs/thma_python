Übung 7.1 (✩)

Gegeben ist folgender Code:

x = 42

if x < 0:
    print('negativ')
elif x == 0:
    print('null')
elif x < 10:
    print('einstellig')
elif x < 100:
    print('zweistellig')
else:
    print('dreistellig oder mehr')

  1. Sagen Sie vorher, welche Ausgabe der Code für x = 42 erzeugt. Notieren Sie Ihre Antwort, bevor Sie den Code ausführen.

  2. Führen Sie den Code aus und überprüfen Sie Ihre Vorhersage.

  3. Welche Ausgabe ergibt sich für die Werte -5, 0, 7 und 1000? Überlegen Sie zuerst, dann ausführen.

  4. Erklären Sie: Warum wird bei x = 7 nur "einstellig" ausgegeben und nicht auch "zweistellig"? Was wäre passiert, wenn statt elif drei unabhängige if-Blöcke verwendet worden wären?

# Code-Zelle
Lösung

  1. Vorhersage für x = 42: "zweistellig", da 42 nicht kleiner als 0, nicht gleich 0 und nicht kleiner als 10 ist, aber kleiner als 100.

  2. Code zur Überprüfung:

x = 42
if x < 0:
    print('negativ')
elif x == 0:
    print('null')
elif x < 10:
    print('einstellig')
elif x < 100:
    print('zweistellig')
else:
    print('dreistellig oder mehr')

  1. Testergebnisse:

    • -5 → "negativ"

    • 0 → "null"

    • 7 → "einstellig"

    • 1000 → "dreistellig oder mehr"

  2. Erklärung: Bei einem if-elif-else-Konstrukt wird die Bedingungsprüfung abgebrochen, sobald eine Bedingung erfüllt ist. Für x = 7 ist die Bedingung x < 10 wahr. Der zugehörige Block wird ausgeführt und alle nachfolgenden elif- und else-Zweige werden übersprungen. Mit drei unabhängigen if-Blöcken wäre sowohl x < 10 als auch x < 100 wahr, es würden also zwei Meldungen ausgegeben werden.

Übung 7.2 (✩)

An einem Druckbehälter wird der Betriebsdruck überwacht. Abhängig vom gemessenen Druck soll eine der folgenden Meldungen ausgegeben werden:

  • Unter 5.0 bar: “Unterdruck: Anlage prüfen.”

  • 5.0 bar bis unter 10.0 bar: “Normalbetrieb.”

  • 10.0 bar bis unter 12.0 bar: “Erhöhter Druck: Beobachten.”

  • Ab 12.0 bar: “Kritisch: Sicherheitsventil öffnen.”

Schreiben Sie ein Skript, das nach dem aktuellen Druck in bar fragt und die entsprechende Meldung ausgibt.

Testen Sie mit den Werten 3.0, 7.5, 11.0 und 13.5 bar.

Strukturieren Sie Ihren Code mit EVA-Kommentaren.

# Code-Zelle
Lösung
# Eingabe
druck_bar = float(input('Aktueller Betriebsdruck in bar: '))

# Verarbeitung und Ausgabe
if druck_bar < 5.0:
    print('Unterdruck: Anlage prüfen.')
elif druck_bar < 10.0:
    print('Normalbetrieb.')
elif druck_bar < 12.0:
    print('Erhöhter Druck: Beobachten.')
else:
    print('Kritisch: Sicherheitsventil öffnen.')

Testergebnisse:

  • 3.0 bar → Unterdruck: Anlage prüfen.

  • 7.5 bar → Normalbetrieb.

  • 11.0 bar → Erhöhter Druck: Beobachten.

  • 13.5 bar → Kritisch: Sicherheitsventil öffnen.

Die Bedingungen können bewusst einfach formuliert werden: Im elif-Zweig für < 10.0 muss nicht geprüft werden, ob der Druck auch >= 5.0 ist, da dieser Fall durch den ersten if-Zweig bereits abgedeckt wurde.

Übung 7.4 (✩✩)

In einer Produktionsanlage darf eine Maschine nur dann in Betrieb genommen werden, wenn alle drei folgenden Bedingungen gleichzeitig erfüllt sind:

  1. Die Öltemperatur liegt zwischen 40 °C und 80 °C.

  2. Der Hydraulikdruck liegt zwischen 3.0 bar und 6.0 bar.

  3. Die Drehzahl liegt zwischen 800 U/min und 1500 U/min.

Schreiben Sie ein Skript, das alle drei Werte abfragt. Wenn alle Bedingungen erfüllt sind, soll ausgegeben werden: "Freigabe erteilt: Maschine kann gestartet werden." Andernfalls: "Freigabe verweigert: Betriebsparameter außerhalb des zulässigen Bereichs."

Testen Sie mit folgenden Wertepaaren:

  • (60 °C, 4.5 bar, 1200 U/min) → sollte Freigabe erteilen

  • (90 °C, 4.5 bar, 1200 U/min) → sollte Freigabe verweigern

  • (60 °C, 4.5 bar, 600 U/min) → sollte Freigabe verweigern

Strukturieren Sie Ihren Code mit EVA-Kommentaren.

# Code-Zelle
Lösung
# Eingabe
oeltemperatur_C = float(input('Öltemperatur in °C: '))
hydraulikdruck_bar = float(input('Hydraulikdruck in bar: '))
drehzahl_umin = float(input('Drehzahl in U/min: '))

# Verarbeitung und Ausgabe
if (40 <= oeltemperatur_C <= 80) and (3.0 <= hydraulikdruck_bar <= 6.0) and (800 <= drehzahl_umin <= 1500):
    print('Freigabe erteilt: Maschine kann gestartet werden.')
else:
    print('Freigabe verweigert: Betriebsparameter außerhalb des zulässigen Bereichs.')

Testergebnisse:

  • 60 °C, 4.5 bar, 1200 U/min → Freigabe erteilt: Maschine kann gestartet werden.

  • 90 °C, 4.5 bar, 1200 U/min → Freigabe verweigert: ...

  • 60 °C, 4.5 bar, 600 U/min → Freigabe verweigert: ...

Bei der and-Verknüpfung muss jede einzelne Teilbedingung wahr sein. Es genügt, dass ein einziger Parameter außerhalb des zulässigen Bereichs liegt, um die Freigabe zu verweigern. Die verkettete Schreibweise 40 <= oeltemperatur_C <= 80 ist eine kompakte Alternative zu (oeltemperatur_C >= 40) and (oeltemperatur_C <= 80).

Schreiben Sie ein Skript, das nach dem Bestellwert in EUR fragt sowie mit input() abfragt, ob der Kunde Premium-Mitglied ist ("ja" oder "nein") und ob es seine erste Bestellung ist ("ja" oder "nein"). Wenn mindestens eine der drei Bedingungen erfüllt ist, soll ausgegeben werden: "Versandkostenfrei!", andernfalls "Versandkosten: 4.99 EUR".

# Verarbeitung und Ausgabe

Übung 7.6 (✩✩)

In einem Lottospiel zieht eine Person eine Zufallszahl zwischen 1 und 49. Gleichzeitig wählt der Spieler seine Glückszahl. Wenn beide Zahlen übereinstimmen, hat der Spieler gewonnen.

  1. Fragen Sie den Benutzer oder die Benutzerin nach seiner oder ihrer Glückszahl (ganze Zahl zwischen 1 und 49).

  2. Simulieren Sie 10000 Lottoziehungen mit np.random.randint().

  3. Zählen Sie, wie viele Ziehungen der Spieler gewonnen hat.

  4. Berechnen Sie die Gewinnquote in Prozent.

  5. Geben Sie die Ergebnisse mit f-Strings aus:

    Glückszahl:     17
Simulationen:   10000
Treffer:           19
Gewinnquote:     0.2 %

  6. Was erwarten Sie theoretisch für die Gewinnquote? Stimmt das simulierte Ergebnis damit überein?

Strukturieren Sie Ihren Code mit EVA-Kommentaren.

# Code-Zelle
Lösung
import numpy as np

# Eingabe
glueckszahl = int(input('Ihre Glückszahl (1 bis 49): '))
ANZAHL_SIMULATIONEN = 10000

# Verarbeitung
anzahl_treffer = 0

for i in range(ANZAHL_SIMULATIONEN):
    gezogene_zahl = np.random.randint(1, 50)   # 1 bis 49
    if gezogene_zahl == glueckszahl:
        anzahl_treffer = anzahl_treffer + 1

gewinnquote = anzahl_treffer / ANZAHL_SIMULATIONEN * 100

# Ausgabe
print(f"Glückszahl:    {glueckszahl:3d}")
print(f"Simulationen:  {ANZAHL_SIMULATIONEN:5d}")
print(f"Treffer:       {anzahl_treffer:5d}")
print(f"Gewinnquote:   {gewinnquote:.1f} %")

Theoretische Gewinnquote: Bei 49 möglichen Zahlen beträgt die Wahrscheinlichkeit eines Treffers 1/49 ≈ 2.04 %. Das simulierte Ergebnis sollte ungefähr in diesem Bereich liegen und mit zunehmender Anzahl Simulationen immer näher daran heranrücken. Diesen Zusammenhang nennt man das Gesetz der großen Zahlen.

Übung 7.7 (✩✩✩)

Bei einem Prüfstandstest werden zwei Messsensoren eingesetzt, die unabhängig voneinander den Zustand einer Maschine als “OK” oder “FEHLER” melden. Wir modellieren den Ausfall eines Sensors als Zufallsexperiment: Ein Sensor meldet mit einer Wahrscheinlichkeit von 10 % einen Fehler. Der Fehler wird simuliert, indem eine Zufallszahl zwischen 1 und 10 erzeugt wird: Zeigt der Würfel eine 1, meldet der Sensor einen Fehler.

Es gelten folgende Regeln:

  • Die Maschine wird abgeschaltet, wenn beide Sensoren einen Fehler melden (redundantes System: erst wenn beide versagen, wird abgeschaltet).

  • Ein Alarm wird ausgelöst, wenn mindestens ein Sensor einen Fehler meldet.

Simulieren Sie 5000 Testdurchläufe und ermitteln Sie:

  1. Wie viele Durchläufe endeten mit einer Abschaltung (beide Fehler)?

  2. Wie viele Durchläufe lösten einen Alarm aus (mindestens ein Fehler)?

  3. Wie viele Durchläufe verliefen ohne Ereignis (kein Fehler)?

Geben Sie die absoluten Häufigkeiten und die prozentualen Anteile mit f-Strings tabellarisch aus und visualisieren Sie die Ergebnisse als Balkendiagramm.

Strukturieren Sie Ihren Code mit EVA-Kommentaren.

# Code-Zelle
Lösung
import numpy as np
import plotly.express as px

# Eingabe
ANZAHL_TESTS = 5000

# Verarbeitung
anzahl_abschaltung = 0   # beide Sensoren Fehler
anzahl_alarm = 0         # genau ein Sensor Fehler
anzahl_ok = 0            # kein Fehler

for i in range(ANZAHL_TESTS):
    # Zufallszahl 1 bis 10, Fehler bei Ergebnis 1 (10 % Wahrscheinlichkeit)
    sensor_1 = np.random.randint(1, 11)
    sensor_2 = np.random.randint(1, 11)

    fehler_1 = (sensor_1 == 1)
    fehler_2 = (sensor_2 == 1)

    if fehler_1 and fehler_2:
        anzahl_abschaltung = anzahl_abschaltung + 1
    elif fehler_1 or fehler_2:
        anzahl_alarm = anzahl_alarm + 1
    else:
        anzahl_ok = anzahl_ok + 1

# Prozentualer Anteil
anteil_abschaltung = anzahl_abschaltung / ANZAHL_TESTS * 100
anteil_alarm = anzahl_alarm / ANZAHL_TESTS * 100
anteil_ok = anzahl_ok / ANZAHL_TESTS * 100

# Ausgabe
print(f"Abschaltung:    {anzahl_abschaltung:4d} von {ANZAHL_TESTS}   ({anteil_abschaltung:.1f} %)")
print(f"Alarm:          {anzahl_alarm:4d} von {ANZAHL_TESTS}   ({anteil_alarm:.1f} %)")
print(f"Kein Ereignis:  {anzahl_ok:4d} von {ANZAHL_TESTS}   ({anteil_ok:.1f} %)")

fig = px.bar(x=["Abschaltung", "Alarm", "Kein Ereignis"],
             y=[anzahl_abschaltung, anzahl_alarm, anzahl_ok],
             labels={"x": "Ereignis", "y": "Anzahl Testdurchläufe"},
             title=f"Sensorsimulation: {ANZAHL_TESTS} Testdurchläufe")
fig.show()

Theoretische Erwartungswerte bei einer Fehlerwahrscheinlichkeit von je 10 %:

  • Abschaltung (beide Fehler): 10 % × 10 % = 1 %

  • Alarm (genau ein Fehler): 2 × 10 % × 90 % = 18 %

  • Kein Ereignis: 90 % × 90 % = 81 %

Die Kombination aus and (Abschaltung) und elif-or (Alarm) sorgt dafür, dass jeder Testdurchlauf genau einem der drei Fälle zugeordnet wird.

Übung 7.8 (✩✩✩)

In Kapitel 5 haben wir Bolzen auf Toleranz geprüft. Jetzt kombinieren wir die Toleranzprüfung mit einer präziseren Klassifizierung der Ausschussteile.

Ein Bolzen soll einen Durchmesser von 25.0 mm haben, mit einer Toleranz von ±0.3 mm. Bolzen außerhalb der Toleranz werden klassifiziert:

  • d < 24.5 mm: “grober Unterfehler”

  • 24.5 mm ≤ d < 24.7 mm: “leichter Unterfehler”

  • 24.7 mm ≤ d ≤ 25.3 mm: “iO”

  • 25.3 mm < d ≤ 25.5 mm: “leichter Überfehler”

  • d > 25.5 mm: “grober Überfehler”

Gegeben sind folgende Messwerte:

durchmesser_mm = [24.3, 24.6, 24.8, 25.0, 25.1, 25.35, 25.6, 24.9, 25.4, 24.4]

Schreiben Sie ein Programm, das für jeden Bolzen die Klassifizierung ausgibt und anschließend die Anzahl in jeder Kategorie zählt und tabellarisch ausgibt.

Strukturieren Sie Ihren Code mit EVA-Kommentaren.

# Code-Zelle
Lösung
# Eingabe
durchmesser_mm = [24.3, 24.6, 24.8, 25.0, 25.1, 25.35, 25.6, 24.9, 25.4, 24.4]

# Verarbeitung
anzahl_grober_unter = 0
anzahl_leichter_unter = 0
anzahl_iO = 0
anzahl_leichter_ueber = 0
anzahl_grober_ueber = 0

for d in durchmesser_mm:
    if d < 24.5:
        klasse = 'grober Unterfehler'
        anzahl_grober_unter = anzahl_grober_unter + 1
    elif d < 24.7:
        klasse = 'leichter Unterfehler'
        anzahl_leichter_unter = anzahl_leichter_unter + 1
    elif d <= 25.3:
        klasse = 'iO'
        anzahl_iO = anzahl_iO + 1
    elif d <= 25.5:
        klasse = 'leichter Überfehler'
        anzahl_leichter_ueber = anzahl_leichter_ueber + 1
    else:
        klasse = 'grober Überfehler'
        anzahl_grober_ueber = anzahl_grober_ueber + 1

    print(f"d = {d:.2f} mm  →  {klasse}")

# Ausgabe: Tabelle
print(f"grober Unterfehler:    {anzahl_grober_unter:3d}")
print(f"leichter Unterfehler:  {anzahl_leichter_unter:3d}")
print(f"iO:                    {anzahl_iO:3d}")
print(f"leichter Überfehler:   {anzahl_leichter_ueber:3d}")
print(f"grober Überfehler:     {anzahl_grober_ueber:3d}")

Ausgabe der Einzelklassifizierungen:

d = 24.30 mm  →  grober Unterfehler
d = 24.60 mm  →  leichter Unterfehler
d = 24.80 mm  →  iO
d = 25.00 mm  →  iO
d = 25.10 mm  →  iO
d = 25.35 mm  →  leichter Überfehler
d = 25.60 mm  →  grober Überfehler
d = 24.90 mm  →  iO
d = 25.40 mm  →  leichter Überfehler
d = 24.40 mm  →  grober Unterfehler

Das if-elif-else-Konstrukt stellt sicher, dass jeder Bolzen genau einer Kategorie zugeordnet wird. Die Variable klasse speichert die Klassifizierung und wird erst innerhalb des Schleifendurchgangs bekannt, weshalb sie nicht vor der Schleife initialisiert werden muss.

Übung 7.9 (✩✩✩) Mini-Projekt

In einem Lager werden Pakete automatisch auf ein Förderband sortiert. Jedes Paket wird nach Gewicht und Abmessungen klassifiziert und einem von drei Förderbändern zugewiesen:

  • Band 1 (Kleinsendung): Gewicht unter 2.0 kg und kürzeste Seite unter 20 cm.

  • Band 2 (Standardpaket): Gewicht zwischen 2.0 kg und 10.0 kg (jeweils einschließlich) oder kürzeste Seite zwischen 20 cm und 60 cm (jeweils einschließlich).

  • Band 3 (Sperrgut): Gewicht über 10.0 kg oder kürzeste Seite über 60 cm.

Wenn ein Paket sowohl die Bedingungen für Band 1 als auch für Band 2 erfüllt, soll es Band 1 zugewiesen werden. Wenn es sowohl Band 2 als auch Band 3 erfüllt, soll es Band 3 zugewiesen werden.

Teil 1: Einzelpaket

Schreiben Sie ein Skript, das nach Gewicht und kürzester Seite fragt und das zugehörige Förderband ausgibt.

Testen Sie mit:

  • (1.5 kg, 15 cm) → Band 1

  • (5.0 kg, 30 cm) → Band 2

  • (12.0 kg, 40 cm) → Band 3

  • (1.0 kg, 25 cm) → Band 2 (leicht, aber zu groß für Band 1)

Teil 2: Simulation eines Arbeitstages

Simulieren Sie 200 ankommende Pakete. Erzeugen Sie für jedes Paket zufällige Werte:

  • Gewicht: gleichverteilt zwischen 0.5 kg und 15.0 kg

  • kürzeste Seite: gleichverteilt zwischen 5 cm und 80 cm

Zählen Sie, wie viele Pakete auf jedes Band geleitet werden, und visualisieren Sie das Ergebnis als Balkendiagramm.

Strukturieren Sie Ihren Code mit EVA-Kommentaren.

# Code-Zelle
Lösung

Teil 1: Einzelpaket

# Eingabe
gewicht_kg = float(input('Gewicht des Pakets in kg: '))
seite_cm = float(input('Kürzeste Seite in cm: '))

# Verarbeitung und Ausgabe
if (gewicht_kg < 2.0) and (seite_cm < 20):
    print('Band 1 (Kleinsendung)')
elif (gewicht_kg > 10.0) or (seite_cm > 60):
    print('Band 3 (Sperrgut)')
else:
    print('Band 2 (Standardpaket)')

Testergebnisse:

  • 1.5 kg, 15 cm → Band 1 (Kleinsendung)

  • 5.0 kg, 30 cm → Band 2 (Standardpaket)

  • 12.0 kg, 40 cm → Band 3 (Sperrgut)

  • 1.0 kg, 25 cm → Band 2 (Standardpaket)

Teil 2: Simulation

import numpy as np
import plotly.express as px

# Eingabe
ANZAHL_PAKETE = 200

# Verarbeitung
anzahl_band1 = 0
anzahl_band2 = 0
anzahl_band3 = 0

for i in range(ANZAHL_PAKETE):
    gewicht_kg = np.random.uniform(0.5, 15.0)
    seite_cm = np.random.uniform(5, 80)

    if (gewicht_kg < 2.0) and (seite_cm < 20):
        anzahl_band1 = anzahl_band1 + 1
    elif (gewicht_kg > 10.0) or (seite_cm > 60):
        anzahl_band3 = anzahl_band3 + 1
    else:
        anzahl_band2 = anzahl_band2 + 1

# Ausgabe
print(f"Band 1 (Kleinsendung):   {anzahl_band1:4d} Pakete  ({anzahl_band1/ANZAHL_PAKETE*100:.1f} %)")
print(f"Band 2 (Standardpaket):  {anzahl_band2:4d} Pakete  ({anzahl_band2/ANZAHL_PAKETE*100:.1f} %)")
print(f"Band 3 (Sperrgut):       {anzahl_band3:4d} Pakete  ({anzahl_band3/ANZAHL_PAKETE*100:.1f} %)")

fig = px.bar(x=["Band 1\n(Kleinsendung)", "Band 2\n(Standardpaket)", "Band 3\n(Sperrgut)"],
             y=[anzahl_band1, anzahl_band2, anzahl_band3],
             labels={"x": "Förderband", "y": "Anzahl Pakete"},
             title=f"Paketsortierung: Simulation von {ANZAHL_PAKETE} Paketen")
fig.show()

Dieses Mini-Projekt kombiniert die zentralen Konzepte aus Kapitel 7: die if-elif-else-Struktur stellt sicher, dass jedes Paket genau einem Band zugewiesen wird, and und or formulieren die kombinierten Bedingungen, und np.random.uniform() aus Kapitel 6.2 erzeugt die Zufallspakete. Die Priorität der Bänder ist bewusst in der Reihenfolge der if-elif-else-Zweige codiert: Band 1 wird zuerst geprüft, Band 3 wird durch elif abgedeckt, Band 2 ist der Auffangfall.
